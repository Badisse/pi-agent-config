# Triage

!!! quote "One-liner"
    Triage issues through a state machine driven by triage roles.

## When to use

- After [to-issues](to-issues.md) has created issues
- When reviewing incoming bugs or feature requests
- Preparing issues for an AFK agent

## How it works

```
> /skill:triage
```

Moves issues through the [triage state machine](../reference/triage-state-machine.md):

```
needs-triage → needs-info (waiting on reporter)
             → ready-for-agent (AFK, fully specified)
             → ready-for-human (needs a person)
             → wontfix
```

## Agent briefs

For each `ready-for-agent` issue, the agent writes an **agent brief** — a structured comment that serves as the contract for the autonomous loop. Briefs are:

- **Behavioral** — describes *what*, not *how*
- **Durable** — no file paths or line numbers that will go stale
- **Concrete** — has acceptance criteria

## Out-of-scope knowledge base

The triage skill maintains `.out-of-scope/` — a knowledge base of rejected features. When a similar request comes in later, the agent surfaces the prior decision.

## AI disclaimer

Every comment posted during triage starts with:

```
> *This was generated by AI during triage.*
```
